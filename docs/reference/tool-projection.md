## Kinds

- **agents** ‚Äî Subagent / persona / mode definitions. [csaw projects: YES]
- **hooks** ‚Äî Lifecycle scripts triggered by tool events. [csaw projects: YES ‚Äî planned v0.6.0]
- **ignore** ‚Äî Context exclusion patterns (.cody/ignore, .cursorignore, .tongyiignore). [csaw projects: YES ‚Äî planned v0.6.0]
- **instructions** ‚Äî Always-on project guidance. AGENTS.md, CLAUDE.md, GEMINI.md, .hermes.md, .goosehints. [csaw projects: YES]
- **mcp** ‚Äî MCP server configs. Per-tool schema varies widely. [csaw projects: YES (via per-tool registry files)]
- **memory** ‚Äî Persistent context across sessions. [csaw projects: NEVER for now ‚Äî most memory is session state or user-private. Revisit if a portable memory standard emerges.]
- **rules** ‚Äî Always-on coding/style rules. [csaw projects: YES]
- **settings** ‚Äî General tool config that may contain multiple kinds. [csaw projects: NEVER ‚Äî settings files contain API keys and personal credentials; mounting them to a team source would leak secrets. Documented as reference only.]
- **skills** ‚Äî On-demand procedural workflows (SKILL.md folders, slash commands, prompts, recipes). [csaw projects: YES]
